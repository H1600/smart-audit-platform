项目名称：智能财务审计数据处理平台
一句话定位：本地部署的智能财务审计数据处理平台，支持上传财务 PDF/Excel → OCR 识别 → ETL 清洗 → 标准化 → 入库 → API 接口 → 前端 UI → Docker 部署，全链路复刻岗位核心工作。
一、覆盖技术清单（全覆盖招聘要求）
Python、Pandas、Numpy、OpenPyXL、PyPDF2
OCR：PaddleOCR（财务定制调优）
ETL：数据抽取、清洗、转换、标准化（含简易 XBRL）
后端：FastAPI、RESTful API
数据库：SQLite、SQLAlchemy
工程化：Pytest 单元测试、日志、异常捕获
容器化：Docker 打包部署
财务基础：科目余额表、序时账、勾稽关系校验
前端：浏览器 UI（本地访问）、文件上传、任务管理、结果查看、导出下载
二、项目核心功能（贴合岗位职责）
多格式文件上传：支持扫描 PDF、图片、非标 Excel（合并单元格、异形表头）
财务 OCR 定制识别：精准提取金额、凭证号、科目编码、日期，准确率≥90%
ETL 全自动清洗：删空数据、修正格式、统一科目、去重、校验勾稽关系
数据标准化：转简易 XBRL 格式，适配审计数据规范
数据入库管理：结构化数据存 SQLite，支持查询、导出 Excel
API 接口服务：提供上传、识别、查询接口，对接前端
工程化 + 容器化：单元测试、日志记录、Docker 一键部署
本地部署约束：数据、日志、模型文件默认保留在本机，不依赖公网服务即可完成核心流程

三、API 与数据模型
1. 核心接口
	- POST /api/files/upload：上传 PDF、图片、Excel，返回 file_id 和任务 id
	- POST /api/tasks/{task_id}/run：触发 OCR + ETL 处理
	- GET /api/tasks/{task_id}：查询任务状态、进度、错误信息
	- GET /api/records：按日期、科目、凭证号、金额分页查询结构化数据
	- GET /api/records/{record_id}：查询单条明细及来源定位信息
	- GET /api/reports/{task_id}：获取勾稽校验报告、异常清单
	- GET /api/export/{task_id}：导出 Excel 或简易 XBRL
	- GET /api/health：健康检查
2. 核心数据模型
	- FileUpload：文件名、文件类型、大小、上传时间、存储路径、处理状态
	- TaskJob：任务编号、关联文件、当前步骤、进度、开始时间、结束时间、错误信息
	- OcrResult：页码、原始文本、结构化字段、置信度、来源坐标
	- LedgerRecord：日期、凭证号、科目编码、科目名称、摘要、借方、贷方、余额、来源文件
	- AuditReport：校验规则、通过结果、异常明细、生成时间
	- AccountMapping：原始科目、标准科目、映射规则、启用状态

四、前端 UI 需求
1. 页面结构
	- 首页/工作台：展示系统状态、待处理任务、最近处理记录、异常提醒
	- 文件上传页：支持拖拽上传、批量上传、文件类型校验、上传进度展示
	- 任务详情页：展示 OCR 识别进度、ETL 步骤、勾稽校验结果、处理日志
	- 数据结果页：支持按日期、科目、凭证号、金额筛选查询，表格分页展示
	- 导出页：支持导出 Excel、简易 XBRL 或校验报告
	- 系统设置页：配置 OCR 模型路径、输出目录、日志级别、默认导出格式
2. 交互要求
	- 上传后自动生成处理任务，前端实时展示状态：待处理、识别中、清洗中、已完成、失败
	- 任务失败时给出明确错误信息和重试入口
	- 查询结果支持高亮异常记录、查看来源文件与页码
	- 前端可直接调用本地后端接口，不要求登录即可进入基础功能
3. 视觉与可用性
	- 界面简洁、偏企业工具风格，适合审计场景
	- 重点突出上传、处理进度和结果校验，不做复杂装饰
	- 支持常见桌面浏览器本地访问，默认适配 1366x768 及以上分辨率

五、部署方式
1. 本地部署优先，支持一键启动后端和前端
2. 默认通过 Docker Compose 或本地脚本启动
3. 运行后可通过浏览器访问前端 UI，通过本地 API 完成全部操作
4. 如后续需要，可扩展为离线内网部署，不依赖外部云资源

六、建议验收口径
1. 能在本地成功启动前端 UI、后端 API 和 SQLite 存储
2. 能完成“上传文件 → OCR → ETL → 入库 → 查询 → 导出”的完整闭环
3. UI 可查看任务状态、处理结果、异常信息和导出文件
4. 核心流程在断网环境下仍可运行
